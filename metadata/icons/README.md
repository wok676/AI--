## 1. 一句话气质

> **「清新可信的健康记录工具:墨绿圆角底 + 一片新叶环抱一簇暖橙热量火苗 —— 健康(绿/叶)× 卡路里(火)× 食欲(暖橙),克制极简、任意分辨率清晰。」**

回扣 `UI-DESIGN.md §0.2` 全局气质:清新(绿)、可信(工具感几何形而非花哨插画)、不焦虑(火苗是温暖橙黄而非报警红)。

## 2. 配色(全部取自 `AppColors`,Hex)

| 角色 | 取自 AppColors | Hex | 用在图标的哪里 |
|---|---|---|---|
| 背景主色(深) | `primary` | `#2E7D5B` | 自适应图标背景 / 商店图标底色起点(墨绿) |
| 背景渐变(浅) | `secondary` | `#4E8C7A` | 底色径向渐变外圈,做轻微立体光感 |
| 叶片主体 | `primaryContainer` | `#B6E5CE` | 环抱的新叶(清新浅绿,与深底拉开层级) |
| 叶脉 / 叶片描边 | `onPrimaryContainer` | `#0A3D2A` | 叶脉细线、叶缘暗调(克制,低透明度) |
| 火苗外焰 | `tertiary` | `#E07A3F` | 卡路里火苗外层(暖橙,= 食物点缀色) |

| 火苗芯 / 高光 | `onPrimary` | `#FFFFFF` | 火苗最亮芯点,提亮焦点 |

> 语义:**叶**=健康/天然食材;**火苗**=卡路里/能量燃烧;叶环抱火=「在健康框架内管理热量」。暖橙黄火苗呼应 App 内宏量「碳水/脂肪」点缀色与 FAB 食欲色调,不用报警红,贴合「不焦虑」气质。

## 3. 图形概念与构图

```
   商店图标 / launcher 图标(1024×1024,带墨绿圆角底)
   ┌───────────────────────────┐
   │   墨绿径向渐变圆角方底       │  primary→secondary 轻渐变
   │        ╭───╮               │
   │      ╭─┘ 🔥 └─╮            │  中心:暖橙→琥珀→白 三层火苗
   │     │  新叶环抱 │           │  两片 primaryContainer 叶自下方
   │      ╲       ╱             │  向上环抱火苗,叶脉 onPrimaryContainer
   │        ╰───╯               │
   └───────────────────────────┘
```

- **构图**:中心对称、单一焦点(火苗),视觉重量集中在画面中部偏上,符合小尺寸 launcher 下的辨识度要求。
- **图形元素**:① 两片对称新叶,自底部向上弧形环抱;② 叶间升起一簇三层火苗(外焰暖橙 / 内焰琥珀黄 / 芯白);③ 叶面一条细叶脉(低透明度,克制)。
- **几何绘制**:全部由 PIL 的椭圆、贝塞尔/多边形路径与径向渐变绘制,**零位图素材**,任意分辨率重绘清晰,规避版权与授权问题。
- **极简克制**:不堆细节,无文字、无写实质感;远观=「一团暖光被绿叶托起」,近观可辨叶与火。

## 4. 安全区与圆角规范

| 项 | 规范 | 说明 |
|---|---|---|
| 画布 | 1024 × 1024 px | 商店与 launcher 主图标基准尺寸 |
| 圆角底(`icon.png`) | 圆角半径 ≈ 画布 22.37%(≈229px) | iOS 风格 squircle 近似;Android 由系统遮罩二次裁切,故圆角仅用于 iOS / 商店主图 |
| 图形安全区(`icon.png`) | 图形主体控制在中心 **≈80%** 圆内(直径 ≈820px) | 四角留白,避免 iOS squircle 裁切吃掉图形 |
| 自适应前景安全区(`icon_foreground.png`) | 图形居中,占画布 **≈70%**(直径 ≈716px) | Android adaptive icon 外圈 ~33% 随机被各厂商遮罩(圆/方/水滴)裁切,核心图形必须落在内 66% 安全圆 |
| 启动 logo(`splash.png`) | 512 × 512,图形占 ≈82% | 透明底,居中放于启动页渐变背景之上 |

> Android 自适应图标硬性约束:**前景关键图形不得超出中心 66% 圆**,否则在「圆形」遮罩机型上会被切。本设计图形主体直径 ≈70%,边缘留有余量,安全。

## 5. 自适应图标前景 / 背景拆分(Android Adaptive Icon)

Android 8.0+ 自适应图标 = 背景层 + 前景层,各 108dp(可视区中心 72dp)。本项目拆分:

| 层 | 来源 | 内容 |
|---|---|---|
| **背景层(background)** | 纯色 `primary` `#2E7D5B`(经 `pubspec.yaml` `adaptive_icon_background` 配置) | 品牌墨绿纯底,各厂商遮罩下始终是干净品牌色,无渐变(纯色更稳,避免遮罩边缘渐变断层) |
| **前景层(foreground)** | `app/assets/icon/icon_foreground.png`(透明底) | 仅「新叶环抱火苗」图形,居中占 ~70%,**不含底色**,透明区域露出背景层墨绿 |

- 商店 / iOS 主图标(`icon.png`)= 背景墨绿渐变底 + 同一前景图形「烧录」在一起的完整 1024 方图(自带圆角)。
- Android adaptive = 纯色墨绿背景 + 透明底前景图形,二者由系统在运行时按机型遮罩合成。
- 启动页(`splash.png`)= 仅前景图形(更小内边距,占 82%),透明底,叠在启动页渐变背景上。

---

## 6. 生成方式

图形由 `tools/gen_app_icon.py`(纯 Pillow,零外部图片)绘制并导出:

- `app/assets/icon/icon.png` — 1024×1024,墨绿圆角渐变底 + 图形(商店 & iOS & launcher fallback)。
- `app/assets/icon/icon_foreground.png` — 1024×1024,透明底,仅图形居中 ~70%(Android adaptive 前景)。
- `app/assets/branding/splash.png` — 512×512,透明底,仅图形居中 ~82%(启动页中心 logo)。

运行方式见脚本头部注释。多分辨率切图、iOS AppIcon set、Android mipmap、启动页注入由 `devops-release-manager` 执行 flutter_launcher_icons / flutter_native_splash(见 `app/pubspec.yaml` 配置段),**UI 不跑 flutter 命令**。
