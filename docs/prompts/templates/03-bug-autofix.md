# 模板 03 · Bug 自动化修复(日志 / 截图 / Chrome CDP)

> 用途:阶段 4 Bug 修复。**严禁人工排查**,必须驱动 Agent 自诊断。
> 对应:Bug 自动化修复率。务必把全过程归档到 sessions/。

---

**角色 Role**:你是 qa-debugger。

**目标 Goal**:定位并修复以下 Bug:【填写:现象】。

**证据输入 Evidence**(粘贴/附上,不靠口述):
- 错误日志:【粘贴 stack / server log】
- 复现截图:【附图】
- Chrome CDP / DevTools 抓取:【网络请求、控制台、DOM 状态】
- 复现步骤:【1. … 2. …】

**约束 Constraints**:
- 不允许我手动改代码,由你基于证据自行诊断。
- 修复后必须给出验证方式并复现确认已解决。
- 不得引入硬编码/越权/崩溃风险。

**输出 Output**:
1. 根因分析(基于证据,不臆测)。
2. 修复方案与代码改动。
3. 验证结果(测试/复现)。
4. 修复 commit(§14 格式)。

**验收 Acceptance**:
- [ ] Bug 不再复现。
- [ ] 无回归(相关测试通过)。
- [ ] 诊断→修复全链路已存入 `docs/prompts/sessions/`。
