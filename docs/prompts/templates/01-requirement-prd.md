# 模板 01 · 需求 → PRD 生成

> 用途:阶段 1 需求与设计。驱动 AI 脑暴并生成结构化 PRD。
> 对应:功能完整性的需求基础。

---

**角色 Role**:你是 Architect + 资深产品经理。

**目标 Goal**:基于以下一句话想法,生成完整 PRD:【填写:App 一句话定位】。

**约束 Constraints**:
- 面向移动 App + 后端 + 管理后台三端。
- 功能需可在练手周期内由 AI 实现并上架。
- 所有面向用户文案后续要走 i18n,PRD 中文案用占位 key 标注。

**输出 Output**(写入 `docs/PRD.md`):
1. 产品定位与目标用户。
2. 核心功能列表(MoSCoW 优先级:Must/Should/Could)。
3. 用户角色与权限矩阵(为纵向/横向鉴权打基础)。
4. 关键页面 / 用户流程。
5. 管理后台需管理的实体与增删改查范围。
6. 非功能需求(性能、安全、i18n、上架合规)。

**验收 Acceptance**:
- [ ] 含权限矩阵(支撑 §6 鉴权)。
- [ ] Must 功能可闭环演示。
- [ ] 明确管理后台范围。
