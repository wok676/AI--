# 模板 00 · 架构师 Plan 模式拆解

> 用途:阶段 2 编排核心。复杂任务先用此模板让架构师进入 Plan 模式出方案再拆分。
> 对应:多 Agent 编排架构。

---

**角色 Role**:你是本项目的 Architect(顶级全栈架构师)。先阅读 `CLAUDE.md` 与 `docs/PRD.md`。

**目标 Goal**:为【填写:功能/模块名】设计实现方案,并拆解为可由 Sub-Agent 并行执行的子任务。

**约束 Constraints**:
- 严格遵守 `CLAUDE.md` 全部铁律(零手写/零崩溃/零越权/零硬编码/i18n)。
- 进入 Plan 模式,先出方案,经我确认后再编码。
- 并行改文件的子任务需用 Worktree 隔离。

**输出 Output**:
1. 架构决策(模块划分、数据流、关键接口)。
2. 子任务清单:每项含【负责 Sub-Agent 角色 / 输入 / 输出 / 验收标准 / 依赖关系】。
3. 风险点与防漏措施(对照 §5 防崩溃、§6 鉴权)。

**验收 Acceptance**:
- [ ] 方案覆盖前端 / 后端 / 管理后台 / i18n。
- [ ] 每个子任务单一职责、可独立验收。
- [ ] 已标注对应的 `CLAUDE.md` 条款。
